# ladder-v1 难度阶梯

`ladder-v1` 用五道人工题观察纯文字二维空间规划能力随约束增加时如何退化。五题都只使用公开、确定性的内置规则，模型输出仍是同一种严格 JSON；因此等级差异来自题面本身，而不是提示词格式变化。

| 等级 | 背包 | 可用格 / 物品总面积 | 实例数 | 新增推理要求 | Oracle |
|---|---:|---:|---:|---|---:|
| L1 | 3×3 | 9 / 7 | 5 | 理解左右相邻效果会随宝石旋转 | 16 |
| L2 | 4×4 缺角 | 15 / 16 | 7 | 不可能全放；需在基础攻击、形状和宝石收益间取舍 | 32 |
| L3 | 4×4 | 16 / 16 | 7 | 同时规划近邻效果和可穿透射线，效果方向随物品旋转 | 50 |
| L4 | 5×5 十字缺口 | 19 / 19 | 7 | 主动避开负面邻接；理解被第一个物品截断的射线；最优解不必塞满 | 52 |
| L5 | 5×5 十字缺口 | 19 / 20 | 8 | 同一战戟同时有正负方向，诅咒射线也会旋转；区分穿透射线和阻挡射线，并识别“护盾墙”可能只是诱饵 | 71 |

## 为什么这样分级

- **L1 只测一个局部关系。** 背包有余量，失败通常说明坐标、形状或旋转方向理解有问题。
- **L2 引入全局取舍。** 总面积比可用格多 1，模型必须明确放弃实例；缺角和 L 形让简单按攻击排序不再可靠。
- **L3 引入不同尺度的影响区。** 宝石只看紧邻格，符文看整条射线；旋转形状时必须同步旋转效果。
- **L4 引入符号和遮挡。** 高基础攻击物品会伤害邻居，射线又会在第一个物品处停止；“全部放入”不是最优策略。
- **L5 组合前四级能力。** 一个来源同时产生互相垂直的正、负效果，另有旋转负面射线、穿透正面射线和无属性挡板；模型需要比较挡诅咒、占空间和直接弃物三种策略。

这里的 L1–L5 是**设计难度假设**，不是先验真理。建议先用 `configs/run.ladder.yaml` 对多个模型运行至少 3 个 trial，再检查每级的合法率、平均 Oracle 比例和最优命中率。若相邻两级分布大量重叠，应优先调整题面，而不是按 Oracle 的节点数强行解释难度。

## 重现

```powershell
uv run bbbench suite validate .\suites\ladder-v1.yaml
uv run bbbench run .\configs\run.ladder.yaml --dry-run
uv run bbbench web
```

Web 实验台会自动发现 `ladder-v1` 和 `run.ladder.yaml`。五份场景位于 `scenarios/ladder/`，精确证明与见证位于 `oracles/ladder/`。
